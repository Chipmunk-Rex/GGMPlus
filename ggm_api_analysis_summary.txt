GGM 사이트/API 분석 정리
작성일: 2026-05-10
대상: https://ggm.gondr.net

1. 분석 범위

- 공개 HTML, CSS, 최신 프론트엔드 번들 JS 확인
- Vue SPA 라우터 경로 추출
- axios 호출 기반 API 엔드포인트 추출
- 공개 GET API 및 로그인 필요 GET API 일부 응답 구조 확인
- 사용자가 제공한 네트워크 헤더에서 authorization, cookie, x-xsrf-token, x-spa-token을 추출해 ggm.gondr.net에만 사용
- 계정 상태를 변경할 수 있는 POST/PUT/DELETE 요청과 보상 수령/출석/로그아웃처럼 상태 변경 가능성이 있는 GET 요청은 실행하지 않음
- 로그인 토큰, 세션 쿠키, XSRF 토큰, SPA 토큰 원문은 이 문서에 저장하지 않음


2. 현재 배포 상태

사이트는 Laravel + Vue 2 계열 SPA로 보인다.
서버는 대부분의 웹 경로에 대해 동일한 SPA HTML을 내려주고,
실제 화면 전환은 프론트엔드 라우터가 처리한다.

현재 HTML 엔트리:
- /css/app.css
- /js/app.js?id=1c375f61c9441fc7a0c5f3eeb3026798

이전 분석 당시 JS 번들:
- /js/app.js?id=72ebcf3f9cc4fc41e3e29f2df17882f3

현재 인증 관련 관찰:
- Bearer 토큰만으로는 일부 신규 API가 403을 반환했다.
- 사용자가 제공한 실제 브라우저 네트워크 헤더처럼 Authorization, Cookie, X-XSRF-TOKEN, X-SPA-TOKEN, X-Requested-With를 함께 보냈을 때 로그인 필요 API가 정상 응답했다.
- 따라서 현재 일부 API는 Bearer 토큰 외에도 세션/XSRF/SPA 토큰 조합을 요구하는 것으로 보인다.

4. 새로 확인된 기능/API

4.1 메인 광고 / 추천 학생

GET /api/main-ads
- 메인 화면 광고 목록
- 인증 헤더 조합 포함 시 정상 응답
- 응답 구조:
  - 배열
  - 항목 필드: id, user_id, ad_type, title, message, image, link_url, gold_cost, duration_days, status, start_date, end_date, admin_note, created_at, updated_at, user_name, user_grade

GET /api/main-ads/price-info
- 메인 광고 신청 가격 정보
- 응답 구조:
  - prices
  - max_concurrent
- prices 하위 필드:
  - 1
  - 3
  - 7

POST /api/main-ads
- 메인 광고 신청
- 상태 변경 요청이므로 실행하지 않음
- 전송 필드:
  - ad_type
  - title
  - message
  - link_url
  - duration_days
  - image 선택 첨부
- 성공 시 사용자 gold 차감 로직이 프론트에 있음

GET /api/user/daily-recommended/3
GET /api/user/daily-recommended/12
- 메인 화면 추천 학생 목록
- 응답 구조:
  - 배열
  - 항목 필드: id, name, grade, profile, short_msg, fame_level, job_group, portfolio_count, praise_score, fame_title
- 기존 /api/user/top15/3, /api/user/top15/12 역할 일부를 대체하는 것으로 보임

- 일일 미션 목록
- 응답 구조:

- 미션 항목 필드:

8. 주의사항

- 로그인 토큰, 세션 쿠키, XSRF 토큰, SPA 토큰은 비밀번호와 유사한 민감한 값이다.
- 이 문서에는 토큰 원문을 저장하지 않았다.
- API 응답 중 일부는 이메일, 자기소개, 포트폴리오, 우편, 파일명 등 개인정보성 필드를 반환한다.
- 분석 결과를 외부에 공유할 때는 개인 식별 정보와 토큰을 제거해야 한다.
- 댓글 작성, 투표, 삭제, 구매, 광고 신청, 미션 보상 수령, 우편 발송/수령 같은 작업은 실제 계정 상태를 바꾸므로 실행 직전 별도 확인이 필요하다.
